📘 LCEL (LangChain Expression Language) SHORT NOTES

---

🔹 What is LCEL?

👉 LCEL is a shorthand syntax to combine runnables
👉 It makes pipelines cleaner and more readable

---

🔹 Without LCEL (Traditional Syntax)

```python
chain = RunnableSequence(r1, r2, r3)
```

👉 Meaning:
Input → r1 → r2 → r3 → Output

---

🔹 With LCEL (New Syntax)

```python
chain = r1 | r2 | r3
```

👉 Meaning:
Input → r1 → r2 → r3 → Output

---

🔹 Key Operator

|  → Represents "pass output to next runnable"

Example:
r1 | r2  =  RunnableSequence(r1, r2)

---

🔹 Internal Working

👉 LCEL is just syntactic sugar
👉 Internally converted to RunnableSequence

Example:
r1 | r2 | r3
= RunnableSequence(r1, r2, r3)

---

🔹 Advantages of LCEL

✔ Cleaner syntax
✔ Easy to read and write
✔ Less boilerplate code
✔ Looks like a pipeline

---

🔹 Key Idea

👉 LCEL does NOT change logic
👉 Only simplifies how we write runnables

---

🔥 FINAL SUMMARY

LCEL = Shortcut for RunnableSequence using "|" operator
